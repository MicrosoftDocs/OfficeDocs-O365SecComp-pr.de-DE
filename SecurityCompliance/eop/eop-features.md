# <a name="eop-features"></a>EOP-Funktionen

Die folgende Tabelle enthält eine Liste der Funktionen, die im gehosteten E-Mail-Filterdienst Exchange Online Protection (EOP) verfügbar sind. 
  
> [!TIP]
> Die [Roadmap: Office 365 for Business](https://office.microsoft.com/en-us/products/office-365-roadmap-FX104343353.aspx) ist eine gute Ressource für Informationen zu demnächst verfügbaren neuen Funktionen. Einen umfassenderen Überblick über die Funktionen in den verschiedenen EOP-Abonnementplänen finden Sie unter [Exchange Online Protection-Dienstbeschreibung](https://go.microsoft.com/fwlink/p/?LinkId=320619).

|**Antispamschutz** <br/> ||
|Eingehende Spamerkennung  <br/> |Der eingehende Antispamschutz ist immer aktiviert und kann nicht deaktiviert werden. Sie können benutzerdefinierte Einstellungen über Ihren Verbindungsfilter und die Richtlinien für die Inhaltsfilterung konfigurieren.  <br/> Für Kunden, die eigenständige EOP: standardmäßig die EOP-Inhaltsfilter als Spam erkannten Nachrichten des Empfängers Junk-e-Mail-Ordner senden. |Ausgehende Spamerkennung  <br/> |Ausgehender Anti-Spam-Schutz ist immer aktiviert, wenn Sie den Dienst für das Senden von ausgehenden e-Mails verwenden, wodurch Organisationen zu schützen, die den Dienst und ihre jeweiligen Empfänger verwenden. Ähnlich wie die Filterung eingehender Nachrichten, ausgehende Spamfilterung besteht Verbindungsfilter und Content-Filterung. Die Filterung ausgehender Spamnachrichten Einstellungen nicht konfigurierbar sind, aber es gibt ausgehender Spamnachrichten aufgeführt, die Sie zum Konfigurieren der Admin-Benachrichtigungen für ausgehende Nachrichten von verdächtigen und blockierter verwenden können. Weitere Informationen finden Sie unter [Konfigurieren der Richtlinie für ausgehende Spamnachrichten](../configure-the-outbound-spam-policy.md).<br/> |
|NDR-Rückläuferschutz  <br/> |Weitere Informationen zu NDR-RÜCKLÄUFER finden Sie unter der NDR-RÜCKLÄUFER festlegen im [erweiterten Spamfilterung Optionen](../advanced-spam-filtering-asf-options.md) als auch [rückläufernachrichten und EOP](../backscatter-messages-and-eop.md).  <br/> |
|Massen-E-Mail-Filterung  <br/> |EOP wurde erweitert und Erkennungsmethoden zur Identifizierung von Massen-e-Mails. Sie können den Dienst zum Markieren von Massen-e-Mail-Mails über die Benutzeroberfläche konfigurieren. Sie können auch Transportregeln zum Prinzipalen Filter Bulk e-Mail durch Suchen des Bulk e-Mail-Nachricht Kopfzeile Stempel erstellen. Weitere Informationen zu Massen-e-Mail, finden Sie unter [Was ist der Unterschied zwischen Junk-e-Mail und Massen-e-Mail?](../what-s-the-difference-between-junk-email-and-bulk-email.md) und die zugehörigen untergeordneten Themen.<br/> |
|Sperrlisten für schädliche URLs  <br/> |EOP verwendet mehrere URL-Sperrlisten, die Unterstützung beim Erkennen bekannter schädlicher Links in Nachrichten bieten.  <br/> |
|Antiphishingschutz  <br/> |EOP umfasst 750.000 Domänen bekannter Spammer.  <br/> |

Weitere Informationen finden Sie unter [Konfigurieren von Richtlinien Ihrer Spam-Filter](../configure-your-spam-filter-policies.md).<br/> |
|Konfiguration von Aktionen, die an Nachrichten mit Inhaltsfilterung ausgeführt werden können  <br/> |Es gibt mehrere konfigurierbare Aktionen. Sie können beispielsweise Content gefilterten Nachrichten löschen oder in den Junk-e-Mail-Ordner oder die Quarantäne zu senden. Weitere Informationen finden Sie unter [Konfigurieren von Richtlinien Ihrer Spam-Filter](../configure-your-spam-filter-policies.md).<br/> |
|Konfiguration von erweiterten Optionen beim aggressiveren Filtern von Spam  <br/> |Weitere Informationen finden Sie unter [Konfigurieren Ihrer Spam-Filter-Richtlinien](../configure-your-spam-filter-policies.md) (Dies ist, in dem Sie konfigurieren) und [Erweiterte Optionen für Spamfilterung](../advanced-spam-filtering-asf-options.md) (die finden Sie bestimmte Informationen über die Funktionsweise der einzelnen Optionen).  <br/> |
|Internationale Spamfilterung  <br/> |Sie können zum Filtern von Nachrichten in bestimmten Sprachen geschrieben oder von bestimmten Ländern oder Regionen gesendete EOP konfigurieren. Sie können bis zu 86 verschiedene Sprachen und 250 unterschiedlichen Regionen konfigurieren. Der Dienst wird die konfigurierte-Aktion für vertrauenswürdige Spam angewendet. Weitere Informationen finden Sie unter [Konfigurieren von Richtlinien Ihrer Spam-Filter](../configure-your-spam-filter-policies.md).<br/> |
|Spam über Outlook oder Outlook Web App (OWA) verwalten  <br/> |Administratoren und Endbenutzer können Listen sicherer Absender und Listen blockierter Absender erstellen. Weitere Informationen:  <br/> OWA: Weitere Informationen finden Sie unter [Blockieren oder Zulassen (Junk-E-Mail-Einstellungen)](https://go.microsoft.com/fwlink/p/?LinkId=294862).  <br/> Outlook: Weitere Informationen finden Sie unter [Übersicht über den Junk-E-Mail-Filter](https://go.microsoft.com/fwlink/p/?LinkId=270065).  <br/> Wenn Sie EOP zum Schutz von lokalen Postfächern verwenden, müssen Sie verzeichnissynchronisierung verwenden, um sicherzustellen, dass diese Einstellungen mit dem Dienst synchronisiert werden. Weitere Informationen zum Einrichten von verzeichnissynchronisierung finden Sie unter "verzeichnissynchronisierung zum Verwalten von e-Mail-Benutzer" in [Manage Mail Users in EOP](manage-mail-users-in-eop.md).           |
|Spamübermittlungen über das Add-In "Junk-E-Mail-Berichtsprogramm" für Microsoft Office Outlook  <br/> |Sie können ein Add-in in Outlook herunterladen, die Sie Spamnachrichten an Microsoft zur Analyse senden können. Weitere Informationen zum Herunterladen und verwenden dieses Tool finden Sie unter [Aktivieren des Berichtnachricht-add-ins](https://support.office.com/article/4250c4bc-6102-420b-9e0a-a95064837676).<br/> Wenn Sie Exchange Server 2013 mit EOP verwenden, können Sie auch in OWA zum Senden von Spamnachrichten, mit der rechten Maustaste wie in den [junk-e-Bericht und Phishing-Mails in Outlook im Web ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md)beschrieben.  <br/> |
|Übermitteln von Spam- und Nicht-Spamnachrichten über einen E-Mail-Alias  <br/> |Sie können Spamnachrichten (Junk) und nicht (kein junk) Spamnachrichten an Microsoft per e-Mail senden. Weitere Informationen finden Sie unter [Submit Spam, nicht-Spam und Phishing-Betrug Nachrichten an Microsoft zur Analyse](../submit-spam-non-spam-and-phishing-scam-messages-to-microsoft-for-analysis.md).<br/> |
|Übermitteln von Spam- und Nicht-Spamnachrichten über das OWA-Junk-E-Mail-Berichtsprogramm  <br/> |Sie können Spam- und nicht-Spamnachrichten an Microsoft über OWA Junk-e-Mail-Bericht senden. Weitere Informationen finden Sie unter [junk-e-Bericht und Phishing-Mails in Outlook im Web ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).<br/> Diese Funktion ist zurzeit für Outlook Web App (OWA)-Kunden verfügbar, deren Exchange Server 2013 SP1-Postfächer mithilfe von EOP gefiltert werden. Exchange Online OWA-Kunden steht diese Funktionalität in naher Zukunft auch zur Verfügung.           |

Weitere Informationen finden Sie unter [Find and release quarantined messages as an administrator](../find-and-release-quarantined-messages-as-an-administrator.md) und [Find and Release Quarantined Messages as an End User](http://technet.microsoft.com/library/e439b560-827a-4807-abd3-6b861c1ff786.aspx).  <br/> |

Weitere Informationen finden Sie unter [Find and release quarantined messages as an administrator](../find-and-release-quarantined-messages-as-an-administrator.md) und [Find and Release Quarantined Messages as an End User](http://technet.microsoft.com/library/e439b560-827a-4807-abd3-6b861c1ff786.aspx).  <br/> |

|**Schutz vor Schadsoftware** <br/> ||
|Schadsoftwareschutz durch Einsatz mehrerer Module  <br/> |Durch mehrere Antischadsoftwaremodule sind unsere Kunden jederzeit automatisch geschützt.  <br/> |
|Option zur Deaktivierung von Schadsoftwarefilterung  <br/> |Sie können die Schadsoftwarefilterung nicht deaktivieren, da wir für alle E-Mails, die über den Dienst weitergeleitet werden, eine Antischadsoftwareprüfung durchführen möchten. Wir sind der Ansicht, dass ein konsistentes und strenges Sicherheitsniveau für alle Kunden ein wichtiger Bestandteil der notwendigen tiefgreifenden Verteidigung zum Schutz Ihrer E-Mail-Umgebung ist. Infolgedessen ist die Schadsoftwarefilterung für alle Kunden automatisch aktiviert.  <br/> |

Diese Benachrichtigungen werden nur gesendet, wenn die gesamte Nachricht gelöscht wird. Weitere Informationen finden Sie unter [Configure Anti-Malware Policies](../configure-anti-malware-policies.md).<br/> |
|Option zum Entfernen von Anlagen, wenn Schadsoftware erkannt wird  <br/> |Administratoren können auswählen, ob die gesamte Nachricht löschen oder der Anhang entfernt und eine angepasste Nachricht an die Empfänger gesendet. Weitere Informationen finden Sie unter [Configure Anti-Malware Policies](../configure-anti-malware-policies.md).<br/> |
|Antispywareschutz  <br/> |Antischadsoftwareschutz umfasst Virenschutz und Antispywareschutz.  <br/> |

Benutzerdefinierte Richtlinien immer Vorrang vor der Standardrichtlinie, aber Sie können die Priorität (d. h., der ausgeführten Reihenfolge) Ihrer benutzerdefinierten Richtlinien ändern. |Richtlinienbasierte Filterung und Aktionen  <br/> |Benutzerdefinierte Richtlinien basieren auf Exchange-Transportregeln. Weitere Informationen finden Sie unter [E-Mail-Fluss Regeln (Transportregeln) in Exchange Online Protection](mail-flow-rules-transport-rules-0.md).<br/> |
|Nach Textmustern filtern  <br/> |In Transportregeln können Arrays oder reguläre Ausdrücke zum Abgleich von Text verwendet werden. |Richtlinienregeln pro Domäne  <br/> |Der Bereich einer Transportregel kann an die Absender- und Empfängerdomänennamen, IP-Adressbereiche, Schlüsselwörter für Adressen oder Muster, Gruppenmitgliedschaften und andere Bedingungen angepasst werden. Weitere Informationen finden Sie unter [Transport Rule Conditions (Predicates)](http://technet.microsoft.com/library/04edeaba-afd4-4207-b2cb-51bcc44e483c.aspx).<br/> |

|Richtlinienregelbenachrichtigungen an den Absender senden  <br/> |Sie können Nachrichten ablehnen und über die Aktion **Nachricht mit Erklärung ablehnen** oder **Nachricht ablehnen mit erweitertem Statuscode** einen Unzustellbarkeitsbericht an den Absender senden. |Nachrichten an feste Adressen senden (z. B. Nachricht an eine bestimmte Adresse weiterleiten oder kopieren)  <br/> |Transportregeln können verwendet werden, um Nachrichten umzuleiten, Empfänger auf CC: oder BCC: zu setzen, Empfänger einfach nur hinzuzufügen oder andere Optionen festzulegen. Weitere Informationen finden Sie unter [Use mail protection reports in Office 365 to view data about malware, spam, and rule detections](http://technet.microsoft.com/library/bcef7984-4bfa-4ca8-9fa5-a65af8618f5d.aspx).  <br/> |
