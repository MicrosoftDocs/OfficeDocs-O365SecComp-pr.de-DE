# <a name="overview-of-advanced-ediscovery-preview-in-microsoft-365"></a>Übersicht über erweiterte eDiscovery (Preview) in Microsoft 365

Microsoft hat nur eine Preview-Version des aktualisierten erweiterte eDiscovery Tools veröffentlicht, die auf die vorhandene eDiscovery-Funktionen in Office 365 aufbaut. Diese Vorabversion aufgerufen *Erweiterte eDiscovery (Preview)*, bietet einen End-to-End-Workflow zum beibehalten, erfassen, überprüfen, analysieren und Exportieren von Inhalten, die Ihre Organisation internen und externen Untersuchungen reagiert. 

## <a name="alignment-with-edrm"></a>Ausrichtung mit EDRM

Mit der durch den Verweis Modell EDRM (Electronic Discovery) beschriebenen eDiscovery-Prozess richtet der integrierte Workflow des erweiterten eDiscovery (Preview) aus. 

![Die Electronic Discovery Reference Model (EDRM)](../media/EDRMv1.png)

(Bildquelle von edrm.net. Das Quelle ein Bild wurde unter Creative Commons Zuweisung 3.0 Unported Lizenz zur Verfügung gestellt.)

Auf allgemeiner Ebene, wie erweiterte eDiscovery hier (Preview) unterstützt die EDRM Workflows:

- **Identification** - Wenn Sie potenzielle Personen in einer Untersuchung von Interesse identifiziert haben, können Sie diese hinzufügen als Verwalter (auch als *Daten Verwalter*, da diese Informationen verfügen können, die für die Untersuchung relevant ist) auf einem erweitert (Vorschau) eDiscovery-Fall. Nachdem der Benutzer als Verwalter hinzugefügt werden, ist es einfach, beibehalten, erfassen und Verwaltungsberechtigter Dokumente zu überprüfen.

- **Permanentes** - beibehalten und Schützen von Daten, die für eine Untersuchung relevant sind, erweiterte eDiscovery (Preview) können, die Sie platzieren, eine rechtliche Aufbewahrungspflicht auf die Datenquellen, die mit der Verwalter in dem Fall verknüpft sind. Sie können auch nicht freiheitsentziehenden Daten in der Warteschleife platzieren. Darüber hinaus hat erweiterte eDiscovery (Preview) ein integrierten Communications Workflows, damit können Sie eine rechtliche Aufbewahrungspflicht Verwalter Benachrichtigung senden und ihrer Empfangsbestätigungen nachverfolgen.

- **Auflistung** -, nachdem Sie die Datenquellen für die Untersuchung relevant identifiziert (und beibehalten), können Sie der integrierten Suchfunktion in erweiterten eDiscovery (Preview) zum Suchen und sammeln Livedaten aus den freiheitsentziehenden Datenquellen (und nicht freiheitsentziehenden verwenden Datenquellen, falls zutreffend) für die Groß-/Kleinschreibung relevant sein können.

- **Verarbeitung** - Nachdem Sie alle Daten in Bezug auf die Groß-/Kleinschreibung gesammelt haben, im nächste Schritt wird verarbeitet weitere Prüfung und Analyse. Erweiterte eDiscovery (Preview) bedeutet dies, dass die Compliance-Daten, die Sie in der Auflistung Phase identifiziert Azure Speicherort (bezeichnet ein *Arbeitssatz*), kopiert werden, die Sie eine statische Ansicht der Groß-/Kleinschreibung Daten bereitstellt. 
 
- **Überprüfung** - nachdem ein funktionierendes Daten hinzugefügt wurde festgelegt ist, können Sie bestimmte Dokumente anzeigen und führen Sie zusätzliche Abfragen an  
 
- **Analyse** - erweiterte eDiscovery (Preview) bietet integrierte Analytics-Tools, mit denen können Sie die Daten aus dem Arbeitssatz ausgemerzte, mit denen Sie bestimmt ist nicht für die Untersuchung relevant. Zusätzlich zur Reduzierung des relevanten Daten, zur nächsten Folie gewechselt eDiscovery (Preview) auch rechtliche Prüfung Kosten zu sparen, mit deren Hilfe Sie von Inhalten zum Überprüfungsprozess einfacher und effizienter organisieren hilft.

- **Produktions-** und **Präsentation** – Wenn Sie bereit sind, können Sie Dokumente aus einer Arbeitssatzes für rechtliche Prüfung exportieren. Sie können Dokumente im systemeigenen Format oder in einem Format EDRM angegeben exportieren, damit sie in der Drittanbieter-Überprüfung Applications importiert werden können.

## <a name="managing-custodians"></a>Verwalten von Verwaltungsberechtigten

Verwenden Sie die Registerkarte **Verwalter** hinzufügen und Verwalten von Personen, die Sie als Personen, die Interesse an die Groß-/Kleinschreibung identifiziert haben. Wenn Sie Verwalter hinzufügen, können Sie schnell Verwaltungsberechtigter-bezogene Aktionen wie rechtliche wie auf Verwaltungsberechtigter Datenquellen platzieren, wie ihre Postfach- und OneDrive-Konto ausführen Kommunikation mit Verwalter und Suche Verwaltungsberechtigter Datenquellen das Erfassen von Inhalten um die Groß-/Kleinschreibung relevant ist. Als die Groß-/Kleinschreibung des Fortschritts ist es einfach, Hinzufügen von neuen Verwalter oder release Verwalter von der Groß-/Kleinschreibung. Weitere Informationen finden Sie unter [Arbeiten mit Verwalter in erweiterten eDiscovery (Preview)](managing-custodians.md).

## <a name="managing-legal-hold-notifications"></a>Rechtliche Aufbewahrungspflicht Benachrichtigungen verwalten

Verwenden Sie die Registerkarte **Kommunikation** , um den Prozess der Kommunikation mit der Verwalter im Fall verwalten. Eine rechtliche Aufbewahrungspflicht Bekanntmachung weist Verwalter auf alle Inhalte zu erhalten, die für die Groß-/Kleinschreibung relevant sein können. Rechtliche Teams müssen möglicherweise nachverfolgen, Hinweise erhalten, lesen und vom Verwalter bestätigt wurde. Der Communications-Workflow in erweiterten eDiscovery (Preview) können Sie erstellen und senden anfänglichen Benachrichtigungen, Erinnerungen, Version Hinweise und Eskalationen Wenn Verwalter nicht eine Benachrichtigung über eine Warteschleife zu bestätigen. Weitere Informationen finden Sie unter [Arbeiten mit der Kommunikation in erweiterten eDiscovery (Preview)](managing-custodian-communications.md).

## <a name="managing-content-preservation"></a>Beibehaltung der Content Management

Wenn Sie eine Anfrage ein Verwaltungsberechtigter hinzufügen, müssen Sie die Option, um einem Haltestatus freiheitsentziehenden Daten zu platzieren. Verwenden Sie die die Registerkarte **enthält** , um den Haltestatus erstellt, wenn Sie Verwalter hinzufügen und Verwalten von zusätzlichen Legal verwalten zugeordnet, mit der Groß-/Kleinschreibung enthält; Sie können beispielsweise identifizieren und einen Haltestatus auf-freiheitsentziehenden Datenquellen platzieren. Sie können auch eine Sperre im Fall bearbeiten und eine abfragebasierte Aufbewahrung, damit nur der Inhalt, der die Abfrage stimmt exakt in die Warteschleife gestellt wird stellen; Sie könnten beispielsweise einen Datumsbereich dem Haltestatus hinzufügen, damit beibehalten nur auf Inhalte, die innerhalb eines bestimmten Datums einem Bereich erstellt wurde. Sie können auch Statistiken auf Inhalte, die in der Warteschleife erhalten möchten, entfernen den Haltestatus nach, wenn es nicht mehr für die Groß-/Kleinschreibung relevant ist oder löschen Sie ihn. Weitere Informationen finden Sie unter [Manage Archive in erweiterten eDiscovery (Preview)](managing-holds.md).

## <a name="indexing-custodian-data"></a>Indizierung Verwaltungsberechtigter Daten

Wenn Sie eine Anfrage ein Verwaltungsberechtigter und die entsprechenden freiheitsentziehenden Datenquellen hinzufügen, wird teilweise indizierten Element aus einer Datenquelle Verwaltungsberechtigter erneut (durch ein so genanntes *erweiterte Indizierung*) indiziert. Dadurch können freiheitsentziehenden Inhalte wie Bilder, nicht unterstützte Dateitypen und andere potenziell nicht indizierte Inhalte vollständig durchsucht werden können, bei der Ausführung Suchvorgänge zum Sammeln von Daten, die für die Groß-/Kleinschreibung relevant ist. Verwenden Sie die Registerkarte **Verarbeitung** zum Überwachen des Status von erweiterte Indizierung und Fix Verarbeitungsfehler (mit einem Prozess Calle *Error Remediation*). Weitere Informationen finden Sie unter [Fix Verarbeitungsfehler in erweiterten eDiscovery (Preview)](processing-data-for-case.md).

## <a name="collecting-case-data"></a>Erfassen von Falldaten

Verwenden Sie die Registerkarte **Suchen** Suchvorgänge nach dem direkten freiheitsentziehenden suchen und -freiheitsentziehenden Datenquellen in Office 365 für Inhalte erstellen, für die Groß-/Kleinschreibung relevant ist. Sie können erstellen und Ausführen von abfragebasierter Suchvorgänge (mithilfe von Schlüsselwörtern und Bedingungen), um eine Reihe von e-Mail-Nachrichten zu ermitteln und Dokumente, die für die Groß-/Kleinschreibung und der Sie relevant sind weitere überprüfen und in den nachfolgenden Schritten im Workflow eDiscovery analysieren möchten. Sie können eine oder mehrere Suchvorgänge die Groß-/Kleinschreibung zugeordnete erstellen. Darüber hinaus können Sie mithilfe der Suchfunktion Beispieldokumente Vorschau und Suche Statistiken anzeigen, die Ihnen möglicherweise weiterhelfen verfeinern und Verbessern der Suchergebnisse. Sobald Sie, dass die Suchergebnisse enthalten, die alle Daten für die Groß-/Kleinschreibung von Bedeutung, fügen Sie die Suchergebnissen auf einen Workingset zur späteren Überprüfung, Analysis zufrieden sind und bei Bedarf culling. Weitere Informationen finden Sie unter [Sammeln von Daten für eine Anfrage in erweiterten eDiscovery (Preview)](collecting-data-for-ediscovery.md).

## <a name="reviewing-and-analyzing-case-data"></a>Überprüfen und Analysieren von Daten der Groß-/Kleinschreibung

Verwenden Sie die Registerkarte **legt arbeiten** oder überprüfen und Analysieren der Inhalts, die Sie von live-System gesammelt und Workingset hinzugefügt haben. Ein *Arbeitssatz* ist eine statische Auflistung (mit anderen Worten, offline-Kopien der Filtern) freiheitsentziehenden Datengruppe (und gegebenenfalls-freiheitsentziehenden Daten), die Sie in der vorherigen Phase des Workflows eDiscovery erfasst. Wenn Sie die Suchergebnisse Workingset hinzufügen, wird ein Prozess ausgelöst, die Dateien aus den Containern extrahiert und extrahiert Metadaten extrahiert den Text. Wenn dieser Prozess abgeschlossen ist, erstellt das System einen neuen Index aller Daten von Verwalter gesammelt und des Arbeitssatzes hinzugefügt. Nachdem die Daten des Arbeitssatzes hinzugefügt wurde, können Sie zusätzliche Abfragen, um die Groß-/Kleinschreibung Daten einzuschränken, Daten als Text oder in der natives Dateiformat anzeigen und mit Anmerkungen versehen, Schwärzen ausführen, und Kennzeichnen von Dokumenten in das Arbeiten legen. Darüber hinaus können Sie erweiterte Analytics ausführen, wie beispielsweise Duplizierung des Dokuments identifizieren, e-Mail-threading und Designs. Nachdem Sie die Daten, die nur was für die Groß-/Kleinschreibung relevant ist entfernt haben, können Sie Download-Dokumente direkt herunterladen oder zusammen mit der Dateimetadaten, Anmerkungen und keine Tags zu exportieren. Weitere Informationen finden Sie unter:

  - [Überprüfen Sie Groß-/Kleinschreibung Daten in erweiterten eDiscovery (Preview)](reviewing-data-in-working-set.md)
  - [Analysieren von Daten in einem Arbeitssatz in erweiterten eDiscovery (Preview)](analyzing-data-in-working-set.md)

## <a name="exporting-data-for-review-and-presentation"></a>Exportieren von Daten zur Überprüfung und Präsentation

Nachdem Sie die Daten aus einer Arbeitssatzes exportiert, verwenden Sie die **exportiert** -Registerkarte Verwalten einer Exportauftrag und Herunterladen von Daten aus einer Menge arbeiten. Beim Exportieren einer Arbeitssatzes die Daten in ein Azure Speicherort hochgeladen werden und klicken Sie dann steht in einem lokalen Computer heruntergeladen werden soll. Sie können den Speicherort abrufen, und Speicher bewerten Schlüssel erforderlich sind, um die exportierten Daten auf der Registerkarte **Exporte** herunterladen. Weitere Informationen finden Sie unter [Exportieren Groß-/Kleinschreibung im erweiterten eDiscovery (Preview)](exporting-data-ediscover20.md).

## <a name="managing-jobs"></a>Verwalten der Aufträge

Verwenden Sie die Registerkarte **Aufträge** zum Überwachen von Prozessen für Aufgaben im Zusammenhang mit der Groß-/Kleinschreibung, die Sie initiiert haben, wie die erneute Indizierung, Suchvorgänge und Exporte. Sie können beispielsweise eine neue Suche auf der Registerkarte **Suchvorgänge** erstellen, die eine große Anzahl von Datenquellen enthält. Der Status dieses Prozesses für die Suche wird auf der Registerkarte **Aufträge** angezeigt. Weitere Informationen finden Sie unter [Verwalten von Aufträgen in erweiterten eDiscovery (Preview)](managing-jobs-ediscovery20.md).

Verwenden Sie die Registerkarte **Einstellungen** für Fall geltende Einstellungen zu konfigurieren. Dazu gehört das Hinzufügen von Mitgliedern zu einer Anfrage, schließen oder Löschen einer Anfrage und Konfigurieren der Suche und-Analyse Verhalten. Weitere Informationen finden Sie unter [Configure Groß-/Kleinschreibung Settings in erweiterten eDiscovery (Preview)](configuring-case-settings-ediscovery20.md).
