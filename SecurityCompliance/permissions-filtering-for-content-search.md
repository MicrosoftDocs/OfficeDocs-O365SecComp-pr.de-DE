## <a name="more-information"></a><span data-ttu-id="257b7-285">Weitere Informationen</span><span class="sxs-lookup"><span data-stu-id="257b7-285">More information</span></span>

- <span data-ttu-id="257b7-p143">**Wie funktioniert die Filterung von Suchberechtigungen?** Der Berechtigungsfilter wird der Suchabfrage hinzugefügt, wenn eine Inhaltssuche ausgeführt wird. Der Berechtigungsfilter ist im Wesentlichen mit der Suchabfrage vom booleschen **und-** Operator verknüpft. Angenommen, Sie verfügen über einen Berechtigungsfilter, mit dem Bob alle Suchaktionen für die Postfächer der Mitglieder der Verteilergruppe "Worker" ausführen kann. Anschließend führt Bob eine Inhaltssuche für alle Postfächer in der Organisation mit der Such `sender:jerry@adatum.com`Abfrage aus. Da der Berechtigungsfilter und die Suchabfrage logisch durch einen **and-** Operator kombiniert werden, gibt die Suche jede Nachricht zurück, die von Jerry@adatum.com an jedes Mitglied der Verteilergruppe "Workers" gesendet wird.</span><span class="sxs-lookup"><span data-stu-id="257b7-p143">**How does search permissions filtering work?** The permissions filter is added to the search query when a Content Search is run. The permissions filter is essentially joined to the search query by the **AND** Boolean operator. For example, say you have a permissions filter that allows Bob to perform all search actions on the mailboxes of members of the Workers distribution group. Then Bob runs a Content Search on all mailboxes in the organization with the search query  `sender:jerry@adatum.com`. Because the permissions filter and the search query are logically combined by an **AND** operator, the search will return any message sent by jerry@adatum.com to any member of the Workers distribution group.</span></span> 
    
- <span data-ttu-id="257b7-p144">**Was geschieht, wenn Sie über mehrere Such Berechtigungsfilter verfügen?** In einer Inhalts Suchabfrage werden mehrere Berechtigungsfilter mit **oder** booleschen Operatoren kombiniert. Ergebnisse werden also zurückgegeben, wenn einer der Filter auf true festgelegt ist. In einer Inhaltssuche werden alle Filter (kombiniert mit **oder** -Operatoren) dann mit der Suchabfrage vom **and-** Operator kombiniert. Im vorherigen Beispiel wird mit einem Suchfilter ermöglicht, dass Bob nur die Postfächer der Mitglieder der Verteilergruppe "Workers" durchsucht. Dann erstellen wir einen weiteren Filter, der verhindert, dass Bob Phils Postfach durchsucht ("Mailbox_Alias-ne" Phil "). Und lassen Sie uns auch davon ausgehen, dass Phil ein Mitglied der Gruppe Arbeiter ist. Wenn Bob eine Inhaltssuche (aus dem vorherigen Beispiel) für alle Postfächer in der Organisation ausführt, werden Suchergebnisse für Phils Postfach zurückgegeben, obwohl Sie Filter angewendet haben, um zu verhindern, dass Bob Phils Postfach durchsucht. Der Grund ist, dass der erste Filter, der es Bob ermöglicht, die Arbeitergruppe zu durchsuchen, auf true festgelegt ist. Und da Phil Mitglied der Gruppe Arbeiter ist, kann Bob Phils Postfach durchsuchen.</span><span class="sxs-lookup"><span data-stu-id="257b7-p144">**What happens if you have multiple search permissions filters?** In a Content Search query, multiple permissions filters are combined by **OR** Boolean operators. Let's take the previous example, where a search filter allows Bob to only search the mailboxes of the members of the Workers distribution group. Then we create another filter that prevents Bob from searching Phil's mailbox ("Mailbox_Alias -ne 'Phil'"). And let's also assume that Phil is a member of the Workers group.
